4. The AI Classification Pipeline ‚Äî How It Works

Here's the exact technical flow when a user shares content from Instagram, TikTok, or YouTube to your app:

1
User taps "Share ‚Üí YourApp" from Instagram/TikTok/YouTube
iOS Share Extension or Android Share Sheet receives the URL. Instantly show "Saving..." confirmation and dismiss.
iOS: Share Extension (Swift) / Android: Intent Filter + Activity
2
Fetch URL metadata (online, ~0.5-2s)
Resolve the URL ‚Üí fetch Open Graph tags (og:title, og:image, og:description). For YouTube: use Data API for title + transcript. Cache everything locally.
iOS: LPMetadataProvider / Android: OkHttp + HTML parsing / YouTube Data API v3
3
Image analysis (offline, ~100-300ms)
Feed cached thumbnail to Vision/ML Kit. Get multi-label classification: "beach, ocean, resort, tropical, vacation" or "dress, fashion, woman, studio". Also run saliency to identify the dominant subject.
iOS: VNClassifyImageRequest / Android: ML Kit ImageLabeler
4
Text analysis (offline, ~200-500ms)
Feed caption/description + hashtags to on-device LLM. Prompt: "Given this social media post, extract: category (travel/shopping/recipe/learning/fitness/entertainment/home/other), key entities (location, price, brand, product), and a one-sentence summary." Returns structured Swift/Kotlin object.
iOS: Foundation Models @Generable / Android: Gemini Nano Prompt API
5
Combine signals ‚Üí assign bucket (offline, instant)
Merge image labels + text classification + hashtag signals. If image says "beach, resort" AND text says "travel" ‚Üí high confidence Travel. If signals conflict, use confidence scores. Store with the highest-confidence bucket + second-best alternative.
Custom scoring logic ‚Äî weighted ensemble of image + text signals
6
Store locally + present to user
Save to local DB: URL, cached thumbnail, title, description, AI summary, bucket assignment, extracted metadata (price, location, etc.), source platform, timestamp. User sees item appear in the correct bucket with rich card.
iOS: SwiftData/Core Data / Android: Room / Flutter: Drift/Hive

Total time from share to classified: Under 3 seconds. The user sees confirmation instantly (step 1), and by the time they open the app, the item is already in the right bucket with a summary.

User scrolls TikTok, sees a Bali travel vlog. Taps Share ‚Üí YourApp. On-device AI sees beach imagery + "Bali, Ubud, temples" in caption ‚Üí auto-files to Travel Bucket. Extracts location: "Bali, Indonesia". Later, user saves a YouTube video "Best restaurants in Bali" ‚Üí same bucket, sub-tagged "food". User saves coding tutorials, design tips, language learning content from YouTube and X threads. AI classifies as Learning Bucket. Auto-generates topic tags: "Python", "UI Design", "Spanish". User can track progress: "Watched 3/8 Python tutorials this week."

8. Platform Decision: Native Swift + Kotlin vs Flutter

This is a critical decision given your Flutter background. Here's the honest tradeoff:

Recommendation: Go native Swift (iOS first), then evaluate Kotlin for Android.

The reason is specific to this app: Apple's Foundation Models framework is Swift-only. The @Generable macros, guided generation, and tool calling APIs are deeply integrated with Swift's type system. There is no Objective-C bridge and no Flutter plugin. You'd need to write the entire AI classification pipeline in Swift anyway and bridge it to Flutter via platform channels ‚Äî which adds complexity, debugging pain, and latency.

For this particular app, where on-device AI IS the product, native gives you:

  • Direct access to Foundation Models, Vision, Core ML ‚Äî no bridging overhead
  • SwiftUI for the UI (Liquid Glass design language, native feel)
  • Share Extension in native Swift (Flutter share extensions are fragile)
  • First-class iOS 26 widget + Shortcuts support
  • Better App Store positioning (Apple features native apps more)

For Android (Phase 2): Kotlin with ML Kit GenAI APIs + Gemini Nano Prompt API. The Android AI stack is similarly Kotlin-native. A shared backend (Supabase/Firebase) keeps data in sync across platforms.

If you strongly prefer Flutter: It's possible but you'll need Swift platform channels for all AI features on iOS and Kotlin channels for Android. Budget 30-40% additional development time for the bridging layer. The core UI and data layer can still be Flutter.

9. Strategic Recommendations

Why this, why now

Pocket shut down in July 2025, displacing millions of users. Apple Foundation Models just launched (Sept 2025, iOS 26) ‚Äî you'd be among the first wave of apps using it, which Apple actively promotes. Saver/CentralSave is proving viral demand for mobile social saving with zero AI ‚Äî showing the market wants this but nobody has added intelligence yet. High-end phone users are the perfect early adopters: willing to pay, comfortable with new concepts, and their devices can actually run the AI.

The moat

On-device AI creates a genuine competitive advantage. Cloud-based competitors (mymind, Dewey) face latency, cost, and privacy tradeoffs you don't. As Apple and Google improve their on-device models with each OS release, your app gets better for free ‚Äî without increasing your infrastructure costs. Every user correction to bucket classification trains a better model, creating a personalization flywheel no competitor can replicate without the same on-device approach.

Start with iOS

Apple's Foundation Models framework is more mature, better documented, and Apple actively features apps that use it. The iPhone user base skews toward the willingness-to-pay demographic you want. iOS share extensions work more reliably than Android's. Launch on iOS, build traction, then bring it to Android Pixel/Galaxy users.

Pricing for high-end users

Free tier: 50 saves/month, 3 default buckets. This is enough to hook users. Pro ($5.99/mo or $49.99/year): Unlimited saves, unlimited custom buckets, advanced AI (summaries, entity extraction, natural language search), action states, smart review feed. Family ($8.99/mo): Shared buckets for up to 5 people.

Name ideas

BucketSave, Stash.ai, Sortd, PileUp, SaveKit, Cura, Capsule ‚Äî the name should convey both "saving" and "intelligent organization." Short, memorable, available as .com and on the App Store.
